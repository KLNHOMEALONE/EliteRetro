# Galaxy and System Seeds Implementation Plan

> **Source:** BBC Elite procedural generation algorithms (BBC Micro cassette/disc version)
>
> **Implementation:** C# (.NET 9.0), MonoGame (EliteRetro.Core)

---

## Overview

Elite generates 8 galaxies of 256 systems (2,048 total) from three 16-bit seed values using a Tribonacci ("twist") sequence. All system data — name, economy, government, tech level, population, productivity, radius, species, market prices — is derived deterministically from these seeds.

**Starting seeds (Galaxy 0, System 0 — Tibedied):**
- `s0 = 0x5A4A`, `s1 = 0x0248`, `s2 = 0xB753`

---

## Seed Structure

Three 16-bit seeds stored little-endian (low byte first). Stored at QQ15 (current system) and QQ21 (system 0 of current galaxy):

| Seed | Low byte | High byte |
|------|----------|-----------|
| s0   | QQ15     | QQ15+1    |
| s1   | QQ15+2   | QQ15+3    |
| s2   | QQ15+4   | QQ15+5    |

### Bit Layout

```
   s0_hi    s0_lo    s1_hi    s1_lo    s2_hi    s2_lo
 76543210 76543210 76543210 76543210 76543210 76543210
```

Key bit mappings:
- **Galactic X**: `s1_hi` (0-255)
- **Galactic Y**: `s0_hi >> 1` (0-127, chart is half as tall as wide)
- **Economy**: `s0_hi & 0b111` (bits 0-2)
- **Government**: `(s1_lo >> 3) & 0b111` (bits 3-5)
- **Tech level components**: `s1_hi & 0b11` (bits 0-1)
- **Species human flag**: `s2_lo & 0x80` (bit 7; clear = human)
- **Name length**: `s0_lo & 0x40` (bit 6; set = 4 tokens, clear = 3)
- **Name tokens**: bits 0-4 of `s2_hi` (after each twist)
- **Average radius**: `(s2_hi & 0x0F) + 11` high byte, `s1_hi` low byte
- **Planet spawn z_sign**: `((s0_hi & 0b111) + 6 + FIST_bit0) >> 1`
- **Sun spawn z_sign**: `(s1_hi & 0b111) | 0b10000001` (always negative/behind)

## Twisting (Tribonacci Sequence)

Each galaxy starts from an initial seed. Systems are generated by repeatedly **twisting** the seeds. A twist advances one step in the Tribonacci sequence where each number equals the sum of the preceding three.

### Twist Algorithm (TT54)

```
s0' = s1
s1' = s2
s2' = s0 + s1 + s2   (16-bit wraparound)
```

In-place byte-wise (little-endian):
```
tmp_lo = s0_lo + s1_lo
tmp_hi = s0_hi + s1_hi
s0_lo = s1_lo;  s0_hi = s1_hi
s1_lo = s2_lo;  s1_hi = s2_hi
s2_lo = tmp_lo + s1_lo
s2_hi = tmp_hi + s1_hi
```

### Generating Systems

To get from one system to the next: **twist 4 times**. All 256 systems in a galaxy are derived from the galaxy's seed-0 via repeated 4-twist steps.

### Generating Galaxies

To advance to the next galaxy: **rotate each byte left by 1 bit**. After 8 galactic jumps, seeds return to the original. This means all 8 galaxies are derived from the same initial Tibedied seeds.

## System Data Generation (TT24/TT25)

### Economy (QQ28, 0-7)

Derived from `s0_hi & 0b111`. Bit 2 = type (0=Industrial, 1=Agricultural). Bits 0-1 = prosperity.

| Value | Economy |
|-------|---------|
| 0 | Rich Industrial |
| 1 | Average Industrial |
| 2 | Poor Industrial |
| 3 | Mainly Industrial |
| 4 | Mainly Agricultural |
| 5 | Rich Agricultural |
| 6 | Average Agricultural |
| 7 | Poor Agricultural |

**Constraint:** If government is Anarchy (0) or Feudal (1), force bit 1 of economy → economy cannot be Rich.

```csharp
int economyRaw = s0hi & 0b111;
if (government <= 1) economyRaw |= 0b010; // force not-rich for anarchy/feudal
```

### Government (QQ4, 0-7)

Bits 3-5 of `s1_lo`:

| Value | Government |
|-------|------------|
| 0 | Anarchy |
| 1 | Feudal |
| 2 | Multi-government |
| 3 | Dictatorship |
| 4 | Communist |
| 5 | Confederacy |
| 6 | Democracy |
| 7 | Corporate State |

```csharp
int government = (s1lo >> 3) & 0b111;
```

### Tech Level (QQ5, 0-14, displayed as 1-15)

```
techLevel = flipped_economy + (s1_hi AND %11) + (government / 2)
```

Where `flipped_economy` = 3-bit inversion of economy value, division rounds up (LSR + carry).

```csharp
int flippedEconomy = (~economyRaw) & 0b111;
int techLevel = flippedEconomy + (s1hi & 0b11) + ((government + 1) / 2);
techLevel = Math.Clamp(techLevel, 0, 14);
```

Planet appearance: if bit 1 of tech level is set → crater; otherwise meridian/equator.

### Population (QQ6)

```
population = (techLevel * 4) + economy + government + 1
```

Stored as population × 10 (range 1-71 → 0.1B to 7.1B).

```csharp
int population = (techLevel * 4) + economyRaw + government + 1;
population = Math.Clamp(population, 1, 71);
```

### Gross Productivity (QQ7)

```
productivity = (flipped_economy + 3) * (government + 4) * population * 8
```

Measured in millions of credits. Range: 96 to 62,480 M CR.

```csharp
long productivity = (long)(flippedEconomy + 3) * (government + 4) * population * 8;
```

### Average Radius

```
radius = ((s2_hi AND %1111) + 11) * 256 + s1_hi
```

Range: 2,816 to 6,911 km.

```csharp
int radius = ((s2hi & 0x0F) + 11) * 256 + s1hi;
```

### Species Type

```csharp
if ((s2lo & 0x80) == 0)
{
    species = "Human Colonials";
}
else
{
    // Alien species: up to 3 adjectives + type
    int a1 = (s2hi >> 2) & 0b111;         // bits 2-4: size
    int a2 = (s2hi >> 5) & 0b111;         // bits 5-7: color
    int a3 = (s0hi ^ s1hi) & 0b111;       // bits 0-2 of XOR: descriptor
    int a4 = ((s2hi & 0b11) + a3) & 0b111; // bits 0-1 of s2hi + a3: type

    string adj1 = a1 switch { 0 => "Large ", 1 => "Fierce ", 2 => "Small ", _ => "" };
    string adj2 = a2 switch
    {
        0 => "Green ", 1 => "Red ", 2 => "Yellow ",
        3 => "Blue ", 4 => "Black ", 5 => "Harmless ", _ => ""
    };
    string adj3 = a3 switch
    {
        0 => "Slimy ", 1 => "Bug-Eyed ", 2 => "Horned ",
        3 => "Bony ", 4 => "Fat ", 5 => "Furry ", _ => ""
    };
    string type = a4 switch
    {
        0 => "Rodents", 1 => "Frogs", 2 => "Lizards", 3 => "Lobsters",
        4 => "Birds", 5 => "Humanoids", 6 => "Felines", 7 => "Insects"
    };

    species = adj1 + adj2 + adj3 + type;
}
```

### Galactic Coordinates

```
x = s1_hi  (0-255)
y = s0_hi >> 1  (0-127)
```

### Chart Dot Sizes

- **Long-range**: `s2_lo OR 0b01010000` (always ≥80 → one-pixel dots or two-pixel dashes)
- **Short-range**: `(s2_lo AND 1) + 2 + C_flag` (2, 3, or 4)

### Planet Spawn Position (SOLAR)

```
z_sign = ((s0_hi AND %111) + 6 + FIST_bit0) >> 1
x_sign = ROR(z_sign)   // rotate right through carry
y_sign = x_sign
```

FIST bit 0 shifts out on arrival (legal status improves).

### Sun Spawn Position (SOLAR)

```
z_sign = (s1_hi AND %111) OR %10000001   // always negative (behind player), distance 1-7
x_sign = s2_hi AND %11                    // 0-2 offset
y_sign = x_sign
```

## System Name Generation (cpl routine)

### Algorithm

1. Check bit 6 of `s0_lo`: if set → 4 tokens, else → 3 tokens
2. For each token:
   - Take bits 0-4 of `s2_hi`
   - If zero → skip (no output for this token)
   - Otherwise add 128 → index 129-159 into token table (QQ16)
   - Twist seeds, repeat
3. Original seeds are backed up and restored (twisting is destructive)
4. C flag from final twist feeds into short-range chart star size

### Two-Letter Token Table (QQ16)

Indexed 128-159 (32 entries, index 0/128 = skip):

| Index | Token | Index | Token | Index | Token | Index | Token |
|-------|-------|-------|-------|-------|-------|-------|-------|
| 128 | (skip) | 137 | ES | 146 | EN | 155 | AN |
| 129 | LE | 138 | AR | 147 | BE | 156 | TE |
| 130 | XE | 139 | MA | 148 | RA | 157 | IS |
| 131 | GE | 140 | IN | 149 | LA | 158 | RI |
| 132 | ZA | 141 | DI | 150 | VE | 159 | ON |
| 133 | CE | 142 | RE | 151 | TI | | |
| 134 | BI | 143 | A? | 152 | ED | | |
| 135 | SO | 144 | ER | 153 | OR | | |
| 136 | US | 145 | AT | 154 | QU | | |

Note: Token 143 is "A?" (single-character "A" with a trailing byte). Index 128 (5-bit value 0) is skipped by the cpl routine.

### Example Names

| Name | Length | Token Sequence | Derivation |
|------|--------|---------------|------------|
| Ra | 2 | skip, RA, skip | s0_lo bit6=0, twist0→0, twist1→148, twist2→0 |
| Ara | 3 | A, RA, skip | twist0→143, twist1→148, twist2→0 |
| Lave | 4 | LA, VE, skip | twist0→149, twist1→150, twist2→0 |
| Arexe | 5 | A, RE, XE, skip | s0_lo bit6=1, twist0→143, twist1→142, twist2→130, twist3→0 |

| Tibedied | 8 | TI, BI, DI, ED | s0_lo bit6=1, twist0→151, twist1→147, twist2→141, twist3→152 |

## Market Prices and Availability (QQ23)

### QQ23 Table Structure

Each commodity has 4 bytes:

| Byte | Field | Description |
|------|-------|-------------|
| 0 | `base_price` | Base price (e.g. food=19, textiles=20, narcotics=235) |
| 1 | `econ_factor` + units | Bits 0-4: economic factor value; Bit 7: sign (1=negative); Bits 5-6: units |
| 2 | `base_quantity` | Base availability |
| 3 | `mask` | Price/availability fluctuation mask |

### Economy Scale (QQ28)

| Value | Economy |
|-------|---------|
| 0 | Rich Industrial |
| 1 | Average Industrial |
| 2 | Poor Industrial |
| 3 | Mainly Industrial |
| 4 | Mainly Agricultural |
| 5 | Rich Agricultural |
| 6 | Average Agricultural |
| 7 | Poor Agricultural |

### Price Formula

```
price = (base_price + (random AND mask) + economy * economic_factor) * 4
```

Result is in decicredits (10× displayed price for 1 decimal place).

- **Negative economic factors** → cheaper in Agricultural economies (food, textiles)
- **Positive economic factors** → more expensive in Poor Agricultural systems (narcotics, luxuries)
- **Mask** controls volatility: more bits set = wider price swings

```csharp
public int CalculatePrice(CommodityTemplate commodity, int economyQQ28, byte randomValue)
{
    int econFactor = commodity.EconomicFactor; // signed
    int randomContribution = randomValue & commodity.Mask;
    int price = (commodity.BasePrice + randomContribution + economyQQ28 * econFactor) * 4;
    return price; // decicredits
}
```

### Availability Formula

```
quantity = (base_quantity + (random AND mask) - economy * economic_factor) mod 64
```

If result < 0 → 0.

```csharp
public int CalculateAvailability(CommodityTemplate commodity, int economyQQ28, byte randomValue)
{
    int econFactor = commodity.EconomicFactor;
    int randomContribution = randomValue & commodity.Mask;
    int qty = (commodity.BaseQuantity + randomContribution - economyQQ28 * econFactor) % 64;
    return Math.Max(0, qty);
}
```

## Data Structures

### GalaxySeed

```csharp
public struct GalaxySeed
{
    public ushort W0; // s0
    public ushort W1; // s1
    public ushort W2; // s2

    public GalaxySeed(ushort w0, ushort w1, ushort w2)
    {
        W0 = w0; W1 = w1; W2 = w2;
    }

    /// <summary>
    /// Twist: advance one step in the Tribonacci sequence.
    /// s0' = s1, s1' = s2, s2' = s0 + s1 + s2 (16-bit wraparound)
    /// </summary>
    public void Twist()
    {
        ushort tmpLo = (byte)(W0 + W1);
        ushort tmpHi = (byte)((W0 >> 8) + (W1 >> 8));
        W0 = W1;
        W1 = W2;
        W2 = (ushort)(tmpLo + (tmpHi << 8) + W1);
    }

    /// <summary>
    /// Advance to the next galaxy: rotate each byte left by 1.
    /// </summary>
    public void NextGalaxy()
    {
        W0 = RotateLeftBytePair(W0);
        W1 = RotateLeftBytePair(W1);
        W2 = RotateLeftBytePair(W2);
    }

    private static ushort RotateLeftBytePair(ushort val)
    {
        byte lo = (byte)(val & 0xFF);
        byte hi = (byte)(val >> 8);
        lo = (byte)((lo << 1) | (lo >> 7));
        hi = (byte)((hi << 1) | (hi >> 7));
        return (ushort)((hi << 8) | lo);
    }

    public GalaxySeed Copy() => new GalaxySeed(W0, W1, W2);
}
```

## Galaxy Generation Loop

```csharp
public Galaxy GenerateGalaxy(int galaxyIndex)
{
    var seed = GalaxySeed.Galaxy0System0;

    // Advance to target galaxy
    for (int g = 0; g < galaxyIndex; g++)
    {
        seed.NextGalaxy();
    }

    var systems = new StarSystem[256];
    for (int i = 0; i < 256; i++)
    {
        systems[i] = GenerateSystem(galaxyIndex, i, seed);

        // Twist 4 times to get next system's seeds
        seed.Twist();
        seed.Twist();
        seed.Twist();
        seed.Twist();
    }

    return new Galaxy(galaxyIndex, systems);
}
```

**Why 4 twists?** Spreads seed values across the Tribonacci sequence, ensuring variety between consecutive systems.

## Implementation Steps

1. **GalaxySeed struct** — core data structure + twist/next-galaxy
2. **System data derivation** — TT24/TT25 logic (economy, gov, tech, pop, productivity, radius, species, coords)

5. **Market generator** — QQ23 table + price/availability formulas
6. **StarSystem/Galaxy types** — data model with enums
